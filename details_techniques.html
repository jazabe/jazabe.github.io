Détails techniques

Le projet

Nous avons créé ce site dans le cadre de notre DUT Informatique à l’Université de Lyon 1. Ce projet a été réalisé en équipe par Marie Michel et Romane Berthé. Nous avons choisi de créer un site internet de vente de vin appelé Vin&Spritueux. Ce site internet est construit avec les langages de programmation HTML5 et CSS3.


Les différentes catégories du site:

Notre site internet est composé d’une barre de navigation, d’un footer et de différentes sections composant le body. Ces différentes parties sont responsives en fonction de la taille de la fenêtre.
Par ailleurs dans la barre de navigation, sont situés nos différentes catégories comme le vin blanc, le vin rouge et la fiche technique. Il est possible de cliquer sur chacune des catégories et de se déplacer de rubrique en rubrique.
À partir de la page d'accueil, il est possible de suivre le chemin classique d’un client qui consiste à consulter une page, ajouter un produit au panier, consulter le panier et réaliser le paiement ou bien vider le panier.


La barre de navigation

Pour rendre la barre de navigation responsive, nous avons créé un menu responsive déroulant. L’élément “checkbox” a été utilisé pour permettre d’ouvrir et de fermer le menu lorsque la fenêtre est rétrécie. Une animation a été créée avec l’élément menu-icon et des pseudos éléments before et after. Cela permet la création d’une croix pour fermer la barre recherche lorsque la fenêtre est rétrécie.
Par ailleurs, nous avons utilisé le pseudo classe “hover” avec la couleur grise pour créer une animation et faire apparaître le contour de chaque catégorie.
Le code html et css de la barre de navigation fait partie des autres codes de chaque page pour rendre le site plus homogène.


La page d'accueil

Cette page est composée de différentes sections permettant sa mise en page. Ses sections sont aussi responsives pour permettre la lecture du contenu sur téléphone ou tablette. Nous avons utilisé les requêtes média sur la feuille css. Les différentes requêtes présentes dans média sont mises en application quand la taille de la fenêtre est de moins de 80em.
Nous avons choisi cette limite, car nous avons remarqué que le contenu des différentes pages n’était plus responsive à partir de cette limite.
Par exemple, nous avons changé l’emplacement du titre de la page avec la propriété top si la fenêtre n’est pas en plein écran. Pour les différentes images et textes, nous avons utilisé la propriété row et display flex pour que le changement de taille des fenêtres n’impacte pas la mise en page et reste responsive.
De plus, sur cette page, nous avons choisi de rendre certaines images cliquables pour permettre au client de se déplacer de catégorie en catégorie. Pour cela, nous avons utilisé des liens d’attribut a href.
Nous avons aussi fait le choix de voir apparaître le titre seulement lorsqu’une personne passe sa souris sur l’image. Nous avons utilisé l’élément transition, hover et transform.
Pour afficher les promotions sur la page d’accueil nous avons ajouté un bouton, en savoir plus, qui redirige directement vers l’emplacement sur la page des promotions. Nous avons placé dans l’attribut href l'élément ancre.
Les différents textes et titres sont tous centrés avec la propriété text-align. Nous avons homogénéisé la police du site en important une propriété de font-family.


La page produit

Le code html et css de la barre de navigation et du pied de page sont les mêmes que sur la page d’accueil. Seul l’élément header est plus petit sur la page produit que sur la page accueil. Car nous souhaitions pouvoir voir les produits même lorsque nous sommes positionnés en haut de la page. Pour cela, nous avons réduit la taille du header avec l’élément min-height à 60vh. Sur la page d'accueil, la taille de cet élément est de 100vh.
Par ailleurs, nous avons aussi choisi de mettre une transition de 0.5s chaque fois qu’une personne passe sa souris sur la photo d’un produit. De plus, nous avons défini les images des produits avec un pointeur et des liens d’attribut a href. Cela permet de cliquer sur les images et de se rendre sur la page descriptive du produit.
Pour rendre les images des produits responsives, nous avons défini les lignes sur lesquelles se trouvent les images avec les éléments display et flex-wrap. Ces éléments permettent de garder la mise en page de la page produit même ouverte avec une tablette ou un téléphone.
La police utilisée pour la page produit est la même que sur la page d’accueil. À partir de cette page, il est aussi possible de se rendre sur la page d’accueil avec la bar de navigation.


La page détail produit

La page détaillant le produit a été découpée en deux colonnes. La première colonne est composée de l’image du produit. La deuxième colonne est composée de la fiche technique du produit. Pour mettre côte à côte l’image et le descriptif du produit, nous avons utilisé les éléments display et flex wrap.
Par ailleurs, nous avons ajouté un bouton avec l’attribut href pour permettre d’ajouter l’article au panier. Si le client clique sur ajouter au panier, le bouton permet de l’envoyer directement sur la page panier où on retrouve le panier composé du produit. De plus, si le client passe sa souris sur le bouton ajouter au panier, une animation apparaît. Nous avons utilisé la pseudo-classe,hover, avec un background en couleur et une transition de 1s.
Nous avons aussi ajouté à la page du produit l’élément “input type =”number”. Cet élément permet au client de saisir le nombre de produits qu’il souhaite dans le formulaire.


La page panier

Pour la page panier, nous avons divisé en deux parties la page. La première partie est composée de la table avec le produit, la quantité est du total. La deuxième partie est aussi composée d’un tableau avec le prix de la commande, le coût de l’envoi de la commande et le prix total de la commande. Les trois colonnes sont alignées dans le tableau grâce à l’attribution des propriétés display et flex-wrap à la div info-panier contenant le descriptif du produit.
Nous avons aussi ajouté à la page du panier l’élément “input type =”number”. Cet élément permet au client d’augmenter le nombre de produits qu’il souhaite tout en restant sur la page du panier.
La taille des différents tableaux et leur contenu sont responsive en fonction de la taille de l'onglet. Par exemple, lorsque la taille de la page est réduite, la largeur du tableau composé du total final passe de 490px à 210px avec l’élément max-width.
De plus, nous avons créé un bouton supprimer qui permet de rediriger le client vers la page du panier vide avec le lien d’attribut “a href”.


La page panier vide

Pour la page panier vide nous avons utilisé la même structure de code que la page panier. Cependant, nous avons supprimé la ligne ou se trouver le produit et nous avons ajouté à la place un bouton permettant au client d’ajouter un produit à son panier. Pour ce bouton, nous avons aussi utilisé l’attribut “a href” permettant de renvoyer à une autre page.


La page a propos

La page à propos est composée de deux div dont l’une est composée d’une image et la deuxième comporte la partie texte. Nous avons mis les deux div, right et left, sur la même ligne avec l’élément display. Cette page comporte donc la description de notre équipe. Pour mettre en avant certains mots sur la page, nous avons utilisé les propriétés text-décoration, font-weight et color. Par ailleurs, nous avons mis en page notre texte en utilisant l’élément text-align.
Pour que cette page soit responsive nous avons utilisé sur la page css et dans la requête média “grid-template-columns”.


La section pied de page

Le code html et css du pied de page de notre site internet est le même pour permettre un design homogène. Le pied de page est aussi responsive en fonction du choix de la taille de la fenêtre. Nous avons défini ce pied de page avec des colonnes pour permettre une meilleure mise en page.
